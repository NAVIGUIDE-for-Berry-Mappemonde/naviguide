🌐 **Live app:** [https://csw6hpki.run.complete.dev](https://csw6hpki.run.complete.dev)

## 📖 Overview

NAVIGUIDE is a decision-support system (SADP — *Système d'Aide à la Décision Plaisancière*) built specifically for the **Berry-Mappemonde** offshore circumnavigation. It is **not** an ECDIS — it is a strategic route planner that fuses satellite data, AI agents, and real boat performance polars to give the skipper an honest, data-driven expedition briefing.

Key capabilities:
- **Interactive route map** with anti-shipping logic (avoids commercial traffic lanes)
- **AI expedition briefing** generated by Claude (Anthropic) via a multi-agent pipeline
- **Catamaran simulation mode** — step through each leg with live ETA, bearing, and VMG
- **Polar performance upload** — upload your boat's polar CSV for realistic speed/ETA calculations
- **Copernicus live data** — wind, wave, and current overlays at each waypoint
- **Maritime layers** — ZEE zones, shipping lanes, bathymetry
- **Export** — GeoJSON and KML route export with full waypoint metadata

## 🤖 AI Pipeline

The expedition briefing is generated by a **3-step multi-agent pipeline** (~26 seconds total):

```
POST /api/v1/expedition/plan/berry-mappemonde
         │
         ▼
┌─────────────────────────────┐
│  Agent 1 — Route Intelligence│  AWS Bedrock (Claude 3.5 Sonnet)
│  Anti-shipping score analysis│  ~6 sec
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Agent 3 — Risk Assessment  │  AWS Bedrock (Claude 3.5 Sonnet)
│  Per-waypoint risk scoring  │  ~8 sec
└──────────────┬──────────────┘
               │
               ▼
┌─────────────────────────────┐
│  Briefing — Executive Summary│  Anthropic Claude (claude-opus-4-5)
│  Skipper-grade briefing      │  ~14 sec
└─────────────────────────────┘
```

The frontend caches the briefing in **localStorage (24h TTL)** and displays it instantly on subsequent loads while refreshing in the background.

## 🚀 Deployment

### Prerequisites

- Python 3.10+
- Node.js 18+
- `supervisord`
- `ANTHROPIC_API_KEY` environment variable
- AWS credentials (for Bedrock agents — optional, falls back gracefully)

### Environment Variables

```bash
# naviguide-api
ANTHROPIC_API_KEY=sk-ant-...

# naviguide-app (.env.production)
VITE_API_URL=https://your-domain.com
VITE_ORCHESTRATOR_URL=https://your-domain.com
VITE_POLAR_API_URL=https://your-domain.com
VITE_WEATHER_ROUTING_URL=https://your-domain.com
```

### Build & Start

```bash
# 1. Build the React frontend
cd naviguide-app
npm install
npm run build
# → produces dist/ with content-hashed + timestamped bundle

# 2. Install Python dependencies
pip install -r requirements.txt  # for each service

# 3. Start all services via supervisord
supervisorctl reread
supervisorctl update
supervisorctl start all

# 4. Verify
supervisorctl status
# naviguide-api     RUNNING   pid XXXXX
# naviguide-polar   RUNNING   pid XXXXX
# naviguide-proxy   RUNNING   pid XXXXX
# orchestrator      RUNNING   pid XXXXX
```

### Manual Restart

```bash
supervisorctl restart naviguide-proxy
supervisorctl restart naviguide-api
supervisorctl restart orchestrator
supervisorctl restart naviguide-polar
```

## 🔑 Key Features

### Simulation Mode
Step through each leg of the expedition with the catamaran marker snapping to the route polyline. Displays real-time:
- Current leg (from → to)
- Nautical miles covered / remaining
- ETA in hours
- Bearing (°)
- VMG upwind/downwind (if polar data loaded)

### Anti-Shipping Logic
Routes are scored against commercial shipping density. Segments scoring below 0.70 are flagged for manual review. The routing engine actively avoids high-traffic commercial lanes.

### Polar Upload
Upload your boat's polar CSV (TWA × TWS matrix) to get realistic speed and ETA calculations instead of the default 10-knot estimate.

### AI Agent Chat
In simulation mode, query AI agents per leg for:
- Weather routing advice
- Risk assessment
- Safety recommendations

## ⚠️ Known Limitations

- **Not an ECDIS** — does not replace official charts or provide Under Keel Clearance alerts
- **Briefing cache** — expedition briefing is cached 24h in localStorage; hard-refresh clears it
- **AWS Bedrock** — Agents 1 & 3 require valid AWS credentials; fall back to static analysis if unavailable
- **Playwright testing** — headless browser testing requires system libs (`libatk`, `libgbm`, etc.)
